# BGM生成進捗表示 設計

## 目的

BGM生成中に、CodexがSuperColliderコードを作成している段階と、SuperColliderが音声を検証・再生している段階を区別して表示する。生成完了後はユーザーの追加操作なしで試聴を開始する。

## 対象範囲

- BGM制作画面の生成操作と進捗表示
- 既存の非同期 `generate_music` と `preview_music_draft` の連続実行
- 各段階に対応した日本語エラー表示
- 集中中に音声検証を延期する既存仕様の維持

Rust側の生成処理、SuperColliderコード契約、検証基準、保存形式は変更しない。

## 状態と表示

生成操作は次の状態を持つ。

| 状態 | 表示 | 操作 |
| --- | --- | --- |
| `idle` | `生成する` | 生成開始可能 |
| `coding` | `1/2 SuperColliderをコーディング中…` | 生成ボタンを無効化 |
| `audio` | `2/2 音声を生成・検証中…` | 生成ボタンを無効化 |
| `deferred` | `コード生成完了。音声生成は集中終了後に再開できます。` | 手動試聴を表示 |
| `completed` | `生成と音声検証が完了しました。` | 保存・再試聴可能 |
| `failed` | 失敗した段階に対応するメッセージ | 再生成可能 |

## 処理フロー

1. `生成する`を押すと状態を`coding`にする。
2. 既存の`generate_music`を非同期で呼び、生成されたDraftを受け取る。
3. Draftが`deferred_until_focus_ends`なら`deferred`で終了する。
4. それ以外は状態を`audio`にして、Draft IDを指定して`preview_music_draft`を非同期で呼ぶ。
5. SuperColliderの静的検証、5秒間のミュート検証、試聴開始が成功したらDraftを更新して`completed`にする。
6. Codex側で失敗した場合はコード生成エラー、SuperCollider側で失敗した場合は音声生成・検証エラーを表示する。

既存コマンドの完了を段階の境界とするため、推測タイマーやポーリングは使用しない。

## コンポーネント境界

- `StudioScreen`は現在の進捗状態と表示を管理する。
- `LyraContext.generateTrack`は生成した`MusicDraft`を返す。
- `LyraContext.previewDraft`は対象`MusicDraft`を受け取り、検証済み`MusicDraft`を返す。
- 生成と試聴の順序制御は、依存関数を受け取る小さなクライアント関数へ分離する。
- Rust/Tauriコマンドは既存の非同期実装をそのまま利用する。

## エラー処理

- コード生成失敗後は音声処理を開始しない。
- 音声処理失敗時も生成済みDraftは残し、ユーザーが再試聴できるようにする。
- 再実行時は以前のエラー表示を消す。
- 二重クリックは生成ボタンの無効化とRust側の排他制御で防ぐ。

## テスト

- クライアントの順序制御をTDDで実装する。
- `coding → audio → completed`の順で通知され、生成後に試聴が1回呼ばれることを確認する。
- 集中中の延期Draftでは試聴を呼ばず、`deferred`になることを確認する。
- コード生成失敗と音声処理失敗を別々の状態として確認する。
- クライアントの関連テスト、型検査、Rustテスト、Turbo全体チェックを実行する。
