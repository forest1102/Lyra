# Bun / Turborepo Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace npm workspace orchestration with Bun 1.3.10 and a Turborepo task graph, remove `package-lock.json`, and preserve every Lyra MVP build and verification path.

**Architecture:** Bun owns dependency installation, workspace linking, the lockfile, and process execution. The root `turbo.json` owns task dependencies, persistence, outputs, and caching, while workspace `package.json` files expose only the leaf commands Turborepo requires. A private native-tooling workspace exposes Cargo and SuperCollider commands to the same graph.

**Tech Stack:** Bun 1.3.10, Turborepo 2, Expo 57, Tauri 2, TypeScript 5.9, Vitest 3, Cargo

## Global Constraints

- Do not leave any project-owned `npm run` or `npm install` invocation.
- Delete `package-lock.json`; commit the text `bun.lock` generated by Bun.
- Put task dependencies, cache policy, persistence, inputs, and outputs in root `turbo.json`.
- Keep package scripts as leaf adapters only because Turborepo resolves task commands from same-named package scripts.
- Preserve Japanese Conventional Commit messages.
- Do not cache platform-specific Tauri or SuperCollider artifacts in Turbo.

- [x] **Step 1: Write the failing tooling contract test**

```ts
import { existsSync, readFileSync } from "node:fs";
import { describe, expect, test } from "bun:test";

const json = (path: string) => JSON.parse(readFileSync(path, "utf8"));

describe("repository tooling", () => {
  test("uses only Bun's lockfile and package manager", () => {
    expect(json("package.json").packageManager).toBe("bun@1.3.10");
    expect(existsSync("bun.lock")).toBe(true);
    expect(existsSync("package-lock.json")).toBe(false);
  });

  test("registers the required Turbo tasks", () => {
    const tasks = json("turbo.json").tasks;
    for (const task of ["dev", "build", "test", "typecheck", "check", "fmt:check", "build:desktop", "build:supercollider", "check:supercollider"]) {
      expect(tasks[task]).toBeDefined();
    }
    expect(tasks.dev).toMatchObject({ cache: false, persistent: true });
    expect(tasks["build:desktop"].cache).toBe(false);
    expect(tasks["build:supercollider"].cache).toBe(false);
  });
});
```

- [x] **Step 3: Add the Bun manifests and Turbo task graph**

Root `package.json` keeps the workspaces and shared dependencies, adds `tooling/*`, sets `"packageManager": "bun@1.3.10"`, adds `"turbo": "latest"`, and contains no cross-workspace command chain. Use `workspace:*` for `@lyra/domain` in the client.

```json
  "version": "0.1.0",

    "test": "cargo test --workspace",
    "check": "cargo check --workspace",

```

Create `turbo.json` with `build.dependsOn = ["^build"]`, `dev` persistent/no-cache, cacheable TypeScript outputs, native checks without shared `target` outputs, and no-cache desktop/SuperCollider tasks. Declare SuperCollider runtime path variables in `globalPassThroughEnv`.
